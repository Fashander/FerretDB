# `$or`

The `$or` operator combines multiple query expressions and returns documents that match at least one expression.

The operator accepts an array of query expressions as shown below, and one expression must be satisfied for a document to be included in the result set.

```js
db.runCommand({
  find: '<collection_name>',
  filter: {
    $or: [
      { <query_expression1> },
      { <query_expression2> }
    ]
  }
})
```

A query expression can be any valid query such as:

- `{ <field>: <value> }`
- `{ <field>: { <operator>: <value> } }`
- `{ <field>: /<pattern>/ }`

## Implementation notes and constraints

- The `$or` operator cannot be used directly as a top-level aggregation stage; it must be used within a query context like `$match`.
- For array fields, ensure that the query expressions are structured to match the elements within the array, or use `$elemMatch` for more complex queries.
